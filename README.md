# MCP Scheduler Agent 🤖⏰

LLM агент с планировщиком задач и MCP сервером для установки напоминаний через естественный язык.

## Описание проекта

Этот проект реализует LLM агента, который понимает запросы пользователя на естественном языке и автоматически устанавливает напоминания через MCP сервер с использованием APScheduler и SQLite.

### Основные возможности:
- 📝 Пользователь пишет "напомни через 30 секунд попить кофе"
- 🤖 LLM агент анализирует запрос, определяет сообщение и время
- 🔧 Агент вызывает MCP инструмент `add_reminder`
- ⏰ APScheduler планирует выполнение через указанное время
- 🔔 Уведомление появляется через Streamlit без обновления страницы

## Архитектура

```
reminder_agent/
├── app.py              # Streamlit UI + чат + polling уведомлений
├── agent.py            # LLM агент (вызов модели, обработка tool calls)
├── mcp_server.py       # MCP сервер (инструменты + APScheduler + SQLite)
├── requirements.txt    # Зависимости
├── .env                # API ключи (OPENROUTER_API_KEY)
└── test_agent.py       # Тестовый скрипт
```

## Установка и запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка API ключа

Файл `.env` должен содержать ключ OpenRouter API:

```
OPENROUTER_API_KEY=sk-or-v1-your-api-key
```

### 3. Запуск приложения

```bash
streamlit run app.py
```

### 4. Тестирование

```bash
python test_agent.py
```

## Компоненты

### 1. mcp_server.py
MCP сервер предоставляет следующие инструменты:
- `add_reminder(message, delay_seconds)` - создание напоминания
- `list_tasks(status)` - получение списка задач
- `get_due_tasks()` - задачи для выполнения сейчас
- `mark_done(task_id)` - пометка задачи как выполненной

Использует:
- APScheduler для планирования задач
- SQLite для хранения данных
- MCP для взаимодействия с агентами

### 2. agent.py
LLM агент с подключением к:
- OpenRouter API (модель Qwen 3.6 Plus Preview)
- MCP сервер через stdio_client

Агент автоматически:
- Анализирует запросы пользователя
- Конвертирует время в секунды
- Вызывает соответствующие MCP инструменты
- Формирует ответы пользователю

### 3. app.py
Streamlit UI с:
- Чат-интерфейсом для взаимодействия с агентом
- Фоновым polling для уведомлений (каждую секунду)
- Сайдбаром с активными задачами и статистикой
- Toast уведомлениями при выполнении задач

## Потоки данных

### Создание напоминания:
```
Пользователь → Streamlit → agent.process() → LLM → 
add_reminder() → MCP сервер → APScheduler + SQLite
```

### Уведомление:
```
APScheduler → callback → SQLite update →
Streamlit polling → get_due_tasks() → st.toast()
```

## Использование

### Примеры запросов:
- "напомни через 30 секунд попить кофе"
- "напомни через 2 минуты проверить почту"
- "напомни через 1 час позвонить маме"
- "какие задачи ожидают выполнения?"
- "пометить задачу как выполненную"

### UI элементы:
- 💬 Чат-интерфейс для ввода запросов
- 📋 Сайдбар с активными задачами
- 📊 Статистика выполненных/ожидающих задач
- 🔔 Toast уведомления при выполнении напоминаний

## Ограничения

- Точность уведомления: ±1 секунда
- Максимальная задержка: 24 часа
- UI обновление: polling каждую секунду
- Хранение: SQLite (до 100k задач)

## Технические детали

### LLM модель:
- Используется `qwen/qwen3.6-plus-preview:free` через OpenRouter
- System prompt настроен для строгого использования инструментов

### Планировщик:
- BackgroundScheduler с persistence через SQLiteJobStore
- Автоматическое восстановление задач при старте

### База данных:
- SQLite с таблицами `tasks` и `execution_logs`
- Автоматическое создание таблиц при старте

## Разработка

### Добавление новых инструментов:
1. Добавить метод в `ReminderMCP` класс в `mcp_server.py`
2. Добавить инструмент в метод `get_tools()`
3. Добавить обработку в `handle_tool_call()`

### Изменение LLM поведения:
1. Модифицировать system prompt в `agent.py`
2. Добавить новые примеры обработки запросов

### UI изменения:
1. Модифицировать `app.py` для новых функций
2. Добавить новые компоненты в Streamlit интерфейс

## Тестирование
